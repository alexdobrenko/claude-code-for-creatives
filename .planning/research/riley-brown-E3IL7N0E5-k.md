## Key Frameworks & Mental Models

### 1. "Vibe Coding" Concept
- **Definition**: Creating applications through conversational prompts rather than writing code
- **Positioning**: "Millions of people are vibe coding web apps right now"
- **Platforms**: Lovable, Replit, Vibe Code
- **Implication**: Presents this as a movement/trend, not just a tool

### 2. Iterative Prompt Engineering Pattern
Riley demonstrates a clear workflow:
1. **Initial Feature Dump**: Write out all features comprehensively in first prompt
2. **Enable Backend**: Turn on database/backend features before running
3. **Test & Observe**: Interact with generated app, identify issues
4. **Refinement Prompts**: Make specific styling/UX improvements
5. **Clear History**: Reset context when switching to new feature areas

Example progression:
- First prompt: Comprehensive feature list + design preferences
- Follow-up prompts: "Make X bigger", "Center it with Y", "Think like a designer"
- Context management: "Clear history to get a fresh chat... important to clear the history so that your agent has a very small context window"

### 3. Designer-Mindset Prompting
Frequent directive: **"Think like a designer"**
- "Make them look better. Think like a designer."
- "Make it look elegant"
- "I really want this to look good"

Shows trust in AI's aesthetic judgment when given this framing.

### 4. Progressive Feature Addition
Build in layers, test each layer:
1. Core functionality (theme browsing, copying)
2. Design refinement (sizing, spacing, animations)
3. Backend integration (database tracking)
4. Authentication
5. Monetization considerations (decided against, pivoted to lead magnet)
6. Advanced UX (bottom sheets, animations, deep linking)

### 5. Monetization Pivot
Original idea: Freemium model (5 free themes, 45 paid for $3.99-$10)
**Pivot**: Lead magnet for Vibe Code app
- Shows strategic thinking about product ecosystem
- "This would be a cool lead magnet"
- "We can use this as an ancillary product to Vibe Code"

## Language & Communication Style

### Conversational Tone
- **Casual**: "bro", "Let's go", "This is very cool", "damn"
- **Excited**: Frequent enthusiasm markers ("Oh yeah", "boom", "sick")
- **Stream-of-consciousness**: Talks through thought process in real-time

### Prompting Language Patterns

#### Specificity Levels
**High Detail:**
- "I want the phone mockup that takes up like 60% of the screen"
- "I want you to put a copy button to the right of the phone and make the phone taller and make sure it matches the aspect ratio of an actual iPhone"
- "At the top it should say build on vibe code... search the internet and figure out how to create it so that when I click on that with my phone, it links to the Vibe Code app"

**Delegation to AI:**
- "Really think about how to do that in a way that we could in theory add like a hundred different screens"
- "You to really think about that"
- "Come up with four other ones and make them look really good"
- "Search the internet and figure out how to..."

#### Common Phrases
- "I want you to..." (directive)
- "Make it..." (imperative)
- "Please..." (polite framing)
- "This is pretty juicy prompt here" (meta-commentary)
- "Okay. So..." (transition/recap)

### Error Recovery Language
When things don't work:
- No blame on AI
- Matter-of-fact problem identification
- "Oh, I just realized..." (takes responsibility)
- "I asked my team and apparently..." (seeks external help)
- Clear correction prompts with solutions
